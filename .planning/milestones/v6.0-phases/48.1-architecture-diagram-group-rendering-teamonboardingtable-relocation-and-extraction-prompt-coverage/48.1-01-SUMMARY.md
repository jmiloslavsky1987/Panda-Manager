# Phase 48.1 Plan 01: Architecture Diagram Improvements + Relocation + Extraction Coverage

Architecture diagram group rendering, TeamOnboardingTable relocation, and extraction prompt coverage across all v6.0 entity types.

## One-Liner

Added `integration_group` column for dashed-box grouped rendering in architecture diagram, moved TeamOnboardingTable to Teams tab, fixed E2E workflow layout to group by team, and updated extraction prompt to cover all 6 entity types needed for Phase 50.

## What Was Built

**DB Migration: `integration_group` column**
- New column `integration_group TEXT` on `architecture_integrations` table
- Drizzle schema updated; Zod schemas in POST/PATCH routes accept `integration_group: z.string().nullish()`
- Critical fix: original column name `group` is a PostgreSQL reserved keyword — renamed to `integration_group`

**IntegrationEditModal: Group field**
- Added optional "Group (optional)" text input between Phase and Status fields
- Sends `integration_group: group || null` in payload

**InteractiveArchGraph visual improvements**
- Column width: `w-[180px]` → `w-[220px]`
- Group partitioning: integrations with matching `integration_group` render inside dashed-border box with bold uppercase label; ungrouped render as plain cards above
- Phase-aware column header color strips via `phaseHeaderStyle()` function:
  - Blue: Event Ingest, Alert Intelligence
  - Amber: Incident Intelligence, Knowledge Sources, Real-Time Query Sources, Biggy Capabilities
  - Green: Workflow Automation, Outputs & Actions
  - Zinc: Console / default

**TeamOnboardingTable relocation**
- Removed from `CurrentFutureStateTab.tsx` (Architecture tab)
- Added to `TeamEngagementMap.tsx` (Teams tab Section 4) with local state for optimistic updates
- Removed `onOnboardingUpdate` prop from `WorkflowDiagram` → `CurrentFutureStateTab` chain

**E2E Workflow layout fix (post-UAT)**
- `E2eWorkflowsSection.tsx` refactored from per-workflow cards → team-grouped layout
- Each team gets one card with bold team name header; workflows render side-by-side with `divide-x`
- Workflow name shown as colored track badge (ADR blue, Biggy purple)

**Seed scripts**
- `seed-amex-arch-integrations-full.ts`: 37 AMEX architecture integrations with 2 group clusters (ALERT NORMALIZATION, ON-DEMAND DURING INVESTIGATION)
- `seed-amex-e2e-workflows.ts`: derives ADR + Biggy AI E2E workflows from `team_onboarding_status` rows

**Extraction prompt coverage**
- `worker/jobs/document-extraction.ts`: added `integration_group` to architecture entity, full onboarding status fields to team entity, `focus_area` entity type, `e2e_workflow` entity type
- Phase 50 ROADMAP expanded with 6 specific commit handler gaps

## UAT Results

7/8 tests passed. 1 minor issue (E2E layout) identified and fixed within phase.
