## v4.0 Requirements

### Document Extraction (EXTR)

- [x] **EXTR-01**: Document extraction runs reliably in background without losing progress on browser refresh
- [x] **EXTR-02**: Extraction progress displays percentage complete and current chunk being processed
- [x] **EXTR-03**: Long-running extraction completes successfully even if browser refreshes mid-extraction

### Time Tracking (TIME)

- [x] **TIME-01**: User can access /time-tracking route from main navigation showing all time entries across all projects
- [x] **TIME-02**: Each time entry displays project attribution and user can filter by project
- [x] **TIME-03**: Per-project Time Tracking tab removed from workspace tabs; old /customer/[id]/time redirects

### Overview Workstream (WORK)

- [x] **WORK-01**: Database tables have track column populated with 'ADR' or 'Biggy'; existing projects backfilled
- [x] **WORK-02**: Overview tab renders ADR and Biggy onboarding sections separately with standardized phase models

### Overview Metrics (METR, HLTH, TMLN)

- [x] **METR-01**: Metrics section shows onboarding completion percentage, integration counts by status, validation progress
- [x] **HLTH-01**: Health Dashboard displays overall health indicator, risk count by severity, phase health by workstream, active blocker count
- [x] **TMLN-01**: Milestone timeline renders as visual timeline component (not text list) near top of Overview tab

### Weekly Focus & Integration Tracker (WKFO, OINT)

- [x] **WKFO-01**: Weekly focus section displays 3–5 auto-generated priority bullets refreshed weekly via scheduled BullMQ job
- [x] **WKFO-02**: Circular progress bar appears in weekly focus section tied to meaningful progress data
- [x] **OINT-01**: Integration tracker splits into ADR and Biggy sections with workstream-appropriate type sub-groups

### Test Suite (TEST)

- [ ] **TEST-01**: 6 remaining pre-existing test failures (create-project, launch, extraction-status DB mocks) resolved — **DEFERRED to v6.0**
