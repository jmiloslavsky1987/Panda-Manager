    Update getTasksForProject to return `TaskWithBlockedStatus[]`:

    Strategy: after fetching all tasks for the project, compute is_blocked in memory (no extra query needed — all tasks for the project are already fetched):

    ```typescript
    export async function getTasksForProject(projectId: number): Promise<TaskWithBlockedStatus[]> {
      const rows = await db.select().from(tasks)
        .where(eq(tasks.project_id, projectId))
        .orderBy(tasks.created_at)

      // Build id→status map for blocked_by resolution
      const statusMap = new Map(rows.map(t => [t.id, t.status]))

      return rows.map(t => ({
        ...t,
        is_blocked: t.blocked_by !== null &&
          (statusMap.get(t.blocked_by) ?? 'done') !== 'done',
      }))
    }
    ```

    This is a non-breaking change — Task is a subset of TaskWithBlockedStatus, so existing callers receiving Task[] still compile (structural typing). The return type change from `Task[]` to `TaskWithBlockedStatus[]` means callers get the additional is_blocked field without needing changes themselves, but TypeScript will now correctly type tasks as TaskWithBlockedStatus at call sites.
