CRITICAL — Incomplete-first sort must survive filtering:

MilestonesTableClient currently has sort logic that:
1. Splits milestones into `incomplete` and `complete` arrays
2. Sorts each array by date (sortByDate)
3. Concatenates: `[...sortedIncomplete, ...sortedComplete]`

The new filteredMilestones useMemo MUST preserve this by:
1. Applying URL-param filters to the full `milestones` array first
2. THEN splitting the filtered result into incomplete/complete
3. THEN sorting each half and concatenating

```typescript
// filteredMilestones useMemo — preserves incomplete-first sort
const filteredMilestones = useMemo(() => {
  let result = [...milestones]
  if (statusFilter) result = result.filter(m => m.status === statusFilter)
  if (ownerFilter)  result = result.filter(m => m.owner === ownerFilter)
  if (fromDate) {
    result = result.filter(m => {
      const d = m.target ?? m.date ?? ''
      if (!/^\d{4}-\d{2}-\d{2}/.test(d)) return true  // keep non-ISO rows — don't exclude by date
      return d >= fromDate
    })
  }
  if (toDate) {
    result = result.filter(m => {
      const d = m.target ?? m.date ?? ''
      if (!/^\d{4}-\d{2}-\d{2}/.test(d)) return true  // keep non-ISO rows
      return d <= toDate
    })
  }
  // Preserve incomplete-first order (copy existing sort logic from MilestonesTableClient)
  const incomplete = result.filter(m => m.status !== 'completed')
  const complete   = result.filter(m => m.status === 'completed')
  return [...incomplete.sort(sortByDate), ...complete.sort(sortByDate)]
}, [milestones, statusFilter, ownerFilter, fromDate, toDate])
```
