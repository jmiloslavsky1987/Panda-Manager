3. Build exception rows array with fields:
   - projectId: number
   - projectName: string (customer field)
   - exceptionType: 'overdue' | 'stale' | 'blocker' | 'ownership' | 'dependency'
   - description: string (brief explanation)
   - severity: 1-5 (for sorting)

4. Severity ordering per CONTEXT.md:
   - 1: Open blockers (highest)
   - 2: Overdue milestones
   - 3: Missing ownership
   - 4: Stale updates
   - 5: Unresolved dependencies (lowest)

5. Sort exceptions by severity ASC, then projectName ASC

6. Render as collapsible panel below portfolio table:
   - Panel header: "Exceptions ({count})" with expand/collapse toggle
   - Default: expanded if exceptions exist
   - Each row: project name (linked to /customer/[id]) + exception type badge + description
   - Empty state: "No exceptions — all projects healthy" if no exceptions

7. Exception type badges:
   - Overdue: red badge
   - Stale: yellow badge
   - Blocker: orange badge
   - Ownership: amber badge
   - Dependency: orange badge

8. Description templates:
   - Overdue: "Milestone '{nextMilestone}' was due {X} days ago"
   - Stale: "No updates in {X} days"
   - Blocker: "{count} tasks blocked by dependencies"
   - Ownership: "No project owner assigned"
   - Dependency: "Unresolved cross-project dependencies"

Styling: Use Card component from ui/card, Link from next/link for project names, Badge from ui/badge for exception types.
