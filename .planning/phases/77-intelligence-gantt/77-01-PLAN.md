Create GET /api/projects/[projectId]/exceptions returning a typed JSON array of exception records.

Response type:
```typescript
type ExceptionRecord = {
  type: 'overdue_task' | 'at_risk_milestone' | 'stale_item'
  id: number
  name: string           // task title, milestone name, or truncated description
  reason: string         // e.g. "Overdue 3 days", "At Risk", "Stale 21 days"
  link: string           // absolute path e.g. "/customer/5/tasks"
}
```

Auth: Use requireProjectRole(numericId, 'user') pattern — mirror overview-metrics/route.ts exactly.
Add `export const dynamic = 'force-dynamic'` at top.
Use db.transaction with `SET LOCAL app.current_project_id = ${numericId}`.

Queries to build (raw SQL via `sql` template or drizzle ORM):

1. OVERDUE TASKS: status NOT IN ('done', 'complete') AND due ~ '^[0-9]{4}-[0-9]{2}-[0-9]{2}$' AND due::date < CURRENT_DATE
   - Return: id, title as name, reason = "Overdue N days" (compute from due::date)
   - Link: `/customer/${numericId}/tasks`

2. AT-RISK MILESTONES: status = 'at_risk' OR (status != 'complete' AND date ~ '^[0-9]{4}-[0-9]{2}-[0-9]{2}$' AND date::date < CURRENT_DATE)
   - Return: id, name, reason = "At Risk" or "Overdue N days"
   - Link: `/customer/${numericId}/delivery/milestones`

3. STALE ITEMS: created_at < NOW() - INTERVAL '14 days'
   - Tasks: status NOT IN ('done', 'complete')
   - Actions: status NOT IN ('closed', 'done', 'complete') — use status != 'closed'
   - Risks: status NOT IN ('closed', 'resolved', 'mitigated', 'accepted')
   - Truncate description to 60 chars for actions/risks names
   - Reason: "Stale N days" (compute from created_at)
   - Links: tasks → `/customer/${numericId}/tasks`, actions → `/customer/${numericId}/actions`, risks → `/customer/${numericId}/delivery/risks`

IMPORTANT: tasks has NO updated_at — use created_at for stale detection.
IMPORTANT: actions/risks have last_updated (TEXT, unreliable) and created_at (timestamp) — use created_at.
IMPORTANT: milestones has NO updated_at — milestones are NOT included in stale check.

Combine all results into one array. Sort: overdue_task first, then at_risk_milestone, then stale_item. Within each type sort by severity (most overdue first). Return raw array (no cap at API layer — panel caps at 10).
