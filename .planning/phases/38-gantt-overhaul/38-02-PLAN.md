**Architecture decision:** frappe-gantt renders a single flat SVG. True swim lanes with collapsible headers are NOT possible inside frappe-gantt natively. The approach:
- Render ONE frappe-gantt instance per expanded milestone swim lane (each gets its own `<svg>`)
- Each swim lane is a collapsible accordion section with a header row
- The "Unassigned" lane at the bottom follows the same pattern
- All swim lanes collapsed by default (empty Set for expandedLanes state)
- Milestones with no tasks still render as collapsed accordion headers (no chart rendered when 0 tasks)

**Individual lane chart render:**
Each expanded lane with tasks gets a `<SwimLaneChart>` sub-component that owns its own `svgRef` and `ganttRef`. Pass `tasks` (just this lane's tasks), `viewMode`, and `onDateChange` callback.

**SwimLaneChart component (within same file, not exported):**
```tsx
interface SwimLaneChartProps {
  tasks: GanttTask[]
  viewMode: ViewMode
  onDateChange: (taskId: string, start: Date, end: Date, originalStart: string, originalEnd: string) => void
}

function SwimLaneChart({ tasks, viewMode, onDateChange }: SwimLaneChartProps) {
  const svgRef = useRef<SVGSVGElement>(null)
  const ganttRef = useRef<unknown>(null)

  useEffect(() => {
    if (!svgRef.current || tasks.length === 0) return
    svgRef.current.innerHTML = ''
    ganttRef.current = null

    import('frappe-gantt').then(({ default: Gantt }) => {
      if (!svgRef.current) return
      ganttRef.current = new Gantt(svgRef.current, tasks, {
        view_mode: viewMode,
        date_format: 'YYYY-MM-DD',
        popup_trigger: 'click',
        on_date_change: (task, start, end) => {
          // Find original dates from tasks array before drag
          const original = tasks.find(t => t.id === task.id)
          onDateChange(task.id, start, end, original?.start ?? task.start, original?.end ?? task.end)
        },
      })
    })

    return () => { if (svgRef.current) svgRef.current.innerHTML = '' }
  }, [tasks, viewMode, onDateChange])

  return <div className="overflow-x-auto"><svg ref={svgRef} /></div>
}
```

**Drag save handler (in parent GanttChart):**
```typescript
const handleDateChange = useCallback(async (
  taskId: string,
  start: Date,
  end: Date,
  originalStart: string,
  originalEnd: string
) => {
  const newStart = start.toISOString().split('T')[0]
  const newEnd = end.toISOString().split('T')[0]

  try {
    const res = await fetch(`/api/tasks/${taskId}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ start_date: newStart, due: newEnd }),
    })
    if (!res.ok) {
      throw new Error((await res.json()).error ?? 'Save failed')
    }
    // Silent success — no toast per user decision
  } catch (err) {
    const msg = err instanceof Error ? err.message : 'Failed to save date change'
    toast.error(msg)
    // Note: frappe-gantt does not provide a programmatic rollback API.
    // The bar position is visually incorrect until the user navigates away or refreshes.
    // A future enhancement could call gantt.refresh(tasks) to reset, but that
    // requires access to the specific lane's ganttRef. Per user decision,
    // rollback is best-effort: we show the error toast and the page data
    // remains accurate (DB was not written). router.refresh() is NOT called
    // to avoid disrupting other open lanes.
  }
}, [])
```
