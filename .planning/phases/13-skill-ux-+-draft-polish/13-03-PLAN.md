**Step A — Create DraftEditModal.tsx:**

New file `bigpanda-app/components/DraftEditModal.tsx`. Follow ActionEditModal.tsx exactly for structure. Key differences:
- Modal is controlled by parent (not self-contained): accepts `open: boolean` and `onOpenChange: (open: boolean) => void` props (because the trigger is the entire card div, not a discrete DialogTrigger button)
- Props: `{ draft: Draft; open: boolean; onOpenChange: (v: boolean) => void; onSaved: (updated: Draft) => void; onDismissed: (id: string) => void }`
- Do NOT use `DialogTrigger` — the parent (DraftsInbox) controls `open` state by setting `modalDraft`
- Form fields:
  - Subject: `<input type="text" value={subject} onChange={...} placeholder="Email subject" />`
  - Recipient: `<input type="text" value={recipient} onChange={...} placeholder="Recipient email" />`
  - Content (body): `<textarea value={content} onChange={...} rows={8} className="w-full border rounded p-2 text-sm font-mono resize-y" />`
- DialogFooter contains:
  - "Save" button (primary, submit) — calls PATCH with `{ action: 'edit', subject, content, recipient }`
  - "Dismiss Draft" button (destructive/outline) — calls PATCH with `{ action: 'dismiss' }` on success calls `onDismissed(draft.id)`
  - "Cancel" button (ghost) — calls `onOpenChange(false)`
- On successful save: call `onSaved({ ...draft, subject, content, recipient, updated_at: new Date() })` then `onOpenChange(false)`
- Error state: show inline error string below the form if PATCH returns non-ok
- Use Sonner `toast.success` on save (same pattern as DraftsInbox copy action)
- Import `Draft` type from wherever DraftsInbox currently imports it (likely from a types file or inferred from DB schema — check DraftsInbox.tsx imports)
