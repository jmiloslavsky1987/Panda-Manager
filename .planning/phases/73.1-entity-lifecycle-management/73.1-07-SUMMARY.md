# Phase 73.1 Plan 07: Direct Intent Extraction Pre-Pass Summary

**One-liner:** Pass 0.5 pre-pass detects explicit lifecycle instructions (update/close/remove/merge) before standard entity extraction, fixing Gap 1 where instructional text was mis-classified as decision/history/note types.

## What Was Built

Added direct-intent extraction pre-pass (Pass 0.5) that runs after Pass 0 (pre-analysis) but before Pass 1 (standard entity extraction). This pre-pass asks Claude directly: "Does this document contain explicit instructions to update/close/remove/merge named entities?" and extracts lifecycle intents bypassing standard entity classification.

**Why this matters:** Standard extraction passes classify instructional text like "Merchant cutover decision has been completed" as decision/history/note entity types — all of which are either in SKIP_MATCH_TYPES or missing from entity-matcher. This caused Pass 5 to produce zero proposed changes for instructional documents. The pre-pass captures lifecycle instructions BEFORE entity classification happens, ensuring they reach Pass 5 change detection.

## Implementation Details

### Task 1: Direct Intent Extraction Module

Created `bigpanda-app/worker/jobs/direct-intent-extraction.ts` with:
- `DirectIntentItem` interface: captures intent (update/close/remove/merge), entityType, entityName, proposedFields, sourceExcerpt, reasoning
- `runDirectIntentExtraction()` function: uses Anthropic tool use API with structured schema
- Explicit instruction detection: only extracts lifecycle actions ("Action X is complete", "Remove Y section", "Update Z to in-progress")
- Non-fatal error handling: logs warning and returns empty array on failure

### Task 2: Pipeline Integration

Modified `bigpanda-app/worker/jobs/document-extraction.ts`:
- Added import for `runDirectIntentExtraction` and `DirectIntentItem`
- Inserted Pass 0.5 section after Pass 0 (pre-analysis) completes
- Converts DirectIntentItem to ExtractionItem format:
  - Maps entityName to name/description/title fields
  - Preserves original intent in `_direct_intent` field for Pass 5
  - Sets confidence to 0.85 (high confidence for explicit instructions)
  - Converts proposedFields to string values
- Initializes `allRawItems` with `[...directIntentAsExtractionItems]` so direct intent items are prepended
- Skips PDF documents (text extraction only for now)

## Impact

**Gap closure:** Fixes Gap 1 (critical) — instructional documents no longer produce zero proposed changes in Pass 5. Documents with explicit lifecycle instructions ("Close action X", "Update milestone Y status to completed") now generate proposed changes that surface in the UI for user review.

**Pass 5 behavior:** Change detection receives both:
1. Direct intent items (explicit lifecycle instructions extracted by Pass 0.5)
2. Standard extraction items (entities extracted by Passes 1-4)

This comprehensive input enables Pass 5 to detect changes from instructional text AND from standard entity extraction, fixing the blind spot where explicit instructions were mis-classified.

**Non-breaking:** Pre-pass is non-fatal — extraction pipeline completes even if Pass 0.5 fails. Existing extraction behavior unchanged for documents without explicit instructions.
