## Decisions Made

- **Approve route design:** ProposedChange approval and new item insertion use same endpoint. Simplifies client-side logic and maintains transactional consistency (both operations in single request).
- **Approval UX:** Proposed changes approved by default, user explicitly rejects unwanted changes. Mirrors trust-by-default pattern from ExtractionPreview; reduces clicks for common case (approve all).
- **No confirmation dialogs:** Delete buttons have no confirmation dialog. Consistent with existing BusinessOutcomesSection pattern; optimistic removal with automatic rollback on error provides fast UX with safety net.
- **Close status mapping:** Per-entity-type close status (action→closed, risk→resolved, milestone→completed, workstream→complete, task→completed, focus_area→completed). Aligns with Phase 72 enum normalization; maintains semantic meaning per entity type.
- **Lifecycle handler separation:** updateItemLifecycle/closeItemLifecycle/deleteItemLifecycle are distinct from conflict-resolution deleteItem. Separation clarifies intent (document-driven lifecycle vs conflict-resolution); both write audit logs but have different callers.
