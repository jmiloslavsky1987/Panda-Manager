# Phase 73.1 Plan 06: Pass 5 Gap Closure Summary

**One-liner:** Lowered fuzzy match threshold (0.7→0.5) with compensating Claude confidence filter (0.6→0.75) and added businessOutcome entity type for comprehensive change detection

## What Was Built

Closed two medium-severity gaps in Pass 5 change detection:

1. **Lowered trigram similarity threshold** from 0.7 to 0.5 in `findSimilarEntities()` default parameter
   - Increases recall for merge/rename scenarios with partial name overlap (50-70% similarity)
   - Example: merged workflows "Loyalty ADR Alert-to-Incident" + "Loyalty ADR Production Alert-to-Remediation" → "Loyalty ADR Alert-to-Remediation Workflow (Combined)" scores 0.55 similarity

2. **Raised Claude confidence filter** from 0.6 to 0.75 in `runPass5ChangeDetection()`
   - Compensates for increased false positives from lowered trigram threshold
   - Requires Claude to be MORE certain about match classification (update/close/remove)
   - Maintains precision while improving recall

   - Added `case 'businessOutcome':` to entity-matcher switch querying `business_outcomes` table by `title` field

## Technical Notes

### Precision-Recall Trade-off
Lowering the trigram threshold increases **recall** (catches more legitimate matches) but also increases **false positives** (weaker matches). The compensating confidence filter increase maintains **precision** by requiring Claude to be more certain about the classification before accepting the match.

This is a common information retrieval pattern: cast a wide net (fuzzy match), then apply strict filtering (classification confidence).

### Entity Type Naming Convention
The codebase uses inconsistent naming for entity types:
- EntityType enum: `businessOutcome`, `e2e_workflow`, `focus_area` (mix of camelCase and snake_case)
- Database tables: `business_outcomes`, `e2e_workflows`, `focus_areas` (snake_case)

This plan follows the EntityType enum convention (use what's in the type union). Future work could standardize to snake_case across the board.
