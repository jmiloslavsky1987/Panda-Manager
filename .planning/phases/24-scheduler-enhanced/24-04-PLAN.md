    **CreateJobWizard.tsx** (client component):
    Props: `{ open: boolean; onClose: () => void; onJobCreated: (job: ScheduledJob) => void; initialJob?: ScheduledJob }`

    Internal state: `step` (1|2|3), `skillId`, `jobName`, `scope`, `frequency`, `hour`, `minute`, `dayOfWeek`, `dayOfMonth`, `customCron`, `timezone`, `params`

    Wizard structure follows ProjectWizard.tsx pattern:
    - Radix UI `<Dialog>` with `open` prop and `onOpenChange` → onClose
    - Full-screen: `DialogContent` with `className="max-w-4xl h-[90vh] flex flex-col"`
    - Stepper header: numbered steps "1 Skill • 2 Schedule • 3 Params" — Step 3 label grayed out/hidden when skill has no params (use `getWizardSteps(skillId)` from lib/scheduler-skills)
    - Body: renders current step component with appropriate props
    - Footer: Back / Next / Submit buttons; Next disabled if current step validation fails; Submit only on last step

    Submit handler:
    1. Compute `cron_expression = frequencyToCron(frequency, {hour, minute, dayOfWeek, dayOfMonth, cron: customCron})`
    2. For biweekly/once: `cron_expression = null` (these handled by BullMQ `every` — store frequency in skill_params_json under key `_frequency`)
    3. Build payload: `{ name: jobName, skill_name: skillId, cron_expression, timezone, skill_params_json: {...params, scope}, enabled: true }`
    4. Call `POST /api/jobs` (or `PATCH /api/jobs/[initialJob.id]` in edit mode)
    5. On success: call `onJobCreated(newJob)`, close dialog, show `toast.success('Job created')`
    6. On error: show `toast.error('Failed to save job')`, do NOT close

    Pre-fill in edit mode: when `initialJob` is provided, populate all state fields from it on mount (`useEffect` with `initialJob` dependency).

    Fetch projects for JobParamsStep: in the wizard body, fetch `GET /api/projects` on mount (or when skill with params is selected) to populate the projects list. Use TanStack Query `useQuery` if already available in context, otherwise a simple `useEffect + useState`.
