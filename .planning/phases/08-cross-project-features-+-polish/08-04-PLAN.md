Create bigpanda-app/app/api/knowledge-base/route.ts with two handlers:

**GET /api/knowledge-base**
Query params: `q` (optional free-text search), `project_id` (optional integer filter).

- If `q` is provided and >= 2 chars: delegate to `searchAllRecords({ q, type: 'knowledge_base' })` from lib/queries.ts and return `{ entries: results }` (map SearchResult back to entry shape).
- Otherwise: fetch all rows from knowledge_base, LEFT JOIN projects to get project name. Order by created_at DESC.
- Response: `{ entries: KnowledgeBaseEntry & { project_name: string | null }[] }`

**POST /api/knowledge-base**
Body: `{ title: string, content: string, source_trace?: string, project_id?: number, linked_risk_id?: number, linked_history_id?: number, linked_date?: string }`

Validate: title and content are required (400 if missing).

Insert row, return `{ entry: newRow }` with 201.

If `source_trace` is not provided but `project_id` is, auto-build it:
`source_trace = "Project: {project.customer} | Date: {new Date().toISOString().split('T')[0]}"`
This satisfies KB-03 even when the client doesn't explicitly provide source_trace.

Create bigpanda-app/app/api/knowledge-base/[id]/route.ts with two handlers:

**PATCH /api/knowledge-base/[id]**
Body: partial KnowledgeBaseInsert — any combination of `title`, `content`, `source_trace`, `linked_risk_id`, `linked_history_id`, `linked_date`.

Parse `id` from params as integer. Validate it is a valid number (400 if not).
Run `db.update(knowledgeBase).set(body).where(eq(knowledgeBase.id, id)).returning()`.
Return `{ entry: updatedRow }` with 200. Return 404 if no row was updated.

This is the "link" operation for KB-02: the UI calls PATCH with `{ linked_risk_id: X }` to link an entry to a risk.

**DELETE /api/knowledge-base/[id]**
Parse `id` from params. Run `db.delete(knowledgeBase).where(eq(knowledgeBase.id, id))`. Return 204.

Note: Next.js 15 route params are `Promise<{ id: string }>` — use `const { id } = await params` (consistent with Phase 05.1 pattern for async RSC params).
