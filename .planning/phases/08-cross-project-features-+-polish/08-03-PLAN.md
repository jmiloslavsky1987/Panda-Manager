The function signature:
```typescript
export async function searchAllRecords(params: {
  q: string;
  account?: string;       // project.customer filter (case-insensitive ILIKE)
  type?: string;          // 'actions'|'risks'|'key_decisions'|'engagement_history'|'stakeholders'|'tasks'|'artifacts'|'knowledge_base'
  from?: string;          // ISO date string, inclusive lower bound on date field
  to?: string;            // ISO date string, inclusive upper bound on date field
}): Promise<SearchResult[]>
```

Export SearchResult as a TypeScript interface from queries.ts (or from a shared types file if one exists — prefer queries.ts for co-location).

Implementation strategy: 8 separate SQL UNION arms, each using `search_vec @@ plainto_tsquery('english', $query)`. Use raw `sql` template from drizzle-orm for the tsvector expression.

Each arm selects: id, table literal, section literal (human-readable), project_id, project name via JOIN, customer, date field (pick the most meaningful date per table), title field, snippet field (secondary text truncated via SQL SUBSTRING to 200 chars).

Apply filters in each arm:
- Always: `projects.status = 'active'` (JOIN projects for all tables except knowledge_base)
- knowledge_base: no project filter — includes entries with null project_id; for archived project entries, include them (KB-03 spec: entries from archived projects remain searchable)
- account filter: `projects.customer ILIKE '%' || account || '%'` (skip arm if knowledge_base has no project)
- type filter: only run the matching arm
- from/to filter: apply to the table's date field using `>=` / `<=` string comparison

IMPORTANT: If `q` is empty or < 2 characters, return [] immediately (avoid full-table scans on empty query).

Order results by date DESC, limit 100 total across all arms.

Use Promise.all to run all 8 arms in parallel (or UNION ALL in a single query). Single UNION ALL query is preferred for performance — build it dynamically based on active filters.
