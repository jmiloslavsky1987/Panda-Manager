  //                                                               ^^^^^^^^^^^^^^^^^^ BUG: always 0 files

  // Run extractions sequentially
  ;(async () => {
    for (let idx = 0; idx < fileStatuses.length; idx++) {
      const file = fileStatuses[idx]

The inner `continue` guard on line 66 (`if (!file.artifactId || file.status === 'done') continue`) is DIFFERENT from the outer filter and must NOT be changed. It prevents re-processing files that were already extracted in a prior render cycle (tracked via localStatuses). The outer filter on line 59 only needs to know if there are any files worth trying — status is irrelevant for that gate check.

Fix: Line 59 only:
```typescript
// BEFORE:
const filesToExtract = fileStatuses.filter(f => f.artifactId && f.status !== 'done')

// AFTER:
const filesToExtract = fileStatuses.filter(f => f.artifactId)
```

The inner loop already handles per-file status correctly via localStatuses.
