Create `bigpanda-app/skills/SKILLS-DESIGN-STANDARD.md`. This document governs all skills tab documentation skills (NOT backend processing skills like ai-plan-generator).

Document must cover:

## 1. Scope
- Which skills must comply: all skills displayed in the Skills tab that generate reports, emails, or summaries
- Out of scope: ai-plan-generator, wbs-generate-plan, and other background processing skills that write to the DB

## 2. Required YAML Front-Matter Fields
For each field, document: field name, type, valid values, and purpose:
- `label` — string — Human-readable skill name shown in UI
- `description` — string — One-line description shown in UI under skill card
- `input_required` — boolean (true/false) — Whether user must provide text input before running
- `input_label` — string — Label shown on the input field (e.g. "Transcript"). Use empty string "" when input_required is false
- `schedulable` — boolean — Whether this skill can be triggered on a recurring schedule (used by Phase 65 Project-Scoped Scheduling)
- `error_behavior` — "retry" | "fail" — "retry": attempt once more on failure then fail; "fail": fail immediately on first error

## 3. Example Front-Matter Block
Show a complete valid front-matter block for a skill requiring input and one for a skill that does not.

## 4. Front-Matter Placement
- YAML front-matter block MUST appear at the very top of the file, before any other content
- Opening `---` must be the very first line (no blank lines before it)
- The prompt body begins immediately after the closing `---`

## 5. Prompt Body Conventions
- Use second-person ("You are...") or role framing ("You are an expert...")
- Specify output format (JSON, HTML, markdown, plaintext)
- Include required output structure if JSON (field names and types)
- Do not repeat project context instructions — SkillOrchestrator injects it automatically

## 6. Error and Fallback Behavior
- Explain retry semantics: one automatic retry, then mark skill_run as failed
- Explain fail semantics: immediate failure, no retry
- Guideline: Use "retry" for network-sensitive or LLM-response-sensitive skills; use "fail" for deterministic skills where a retry is unlikely to succeed

## 7. Validation
- Skills tab page validates front-matter on every load
- Skills with missing or invalid front-matter display a "Fix required" badge
- Invalid skills can still be run — badge is informational for developers
