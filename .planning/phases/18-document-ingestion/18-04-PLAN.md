  <behavior>
    - POST body: ApproveRequest (artifactId, projectId, items[], totalExtracted)
    - Validates with Zod: artifactId is number, projectId is number, items is array, each item has entityType + fields + approved boolean
    - For each approved item (approved === true): run conflict check (same dedup key as extract route)
    - If conflict found and no conflictResolution provided: returns 409 { conflicts: [{ itemIndex, existingId, existingRecord }] } — client must re-submit with resolutions
    - If conflictResolution = 'skip': no write, count as skipped
    - If conflictResolution = 'replace': delete existing record, insert new one with source attribution
    - If conflictResolution = 'merge': update existing record's fields, add source_artifact_id + ingested_at
    - For non-conflicting approved items: direct INSERT with source='ingestion', source_artifact_id, ingested_at=now()
    - After all writes: PATCH artifact → update ingestion_log_json and ingestion_status='approved'
    - Returns 200 { written: number, skipped: number, rejected: number }
    - key_decisions and engagement_history are append-only: for these types only INSERT is allowed (no replace/merge that would need UPDATE); if conflict detected on these types, treat as 'skip' automatically
  </behavior>
