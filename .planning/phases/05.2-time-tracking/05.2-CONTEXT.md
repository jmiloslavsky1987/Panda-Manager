# Phase 5.2: Time Tracking — Context

**Gathered:** 2026-03-23
**Status:** Ready for planning
**Source:** In-conversation requirements (TIME-01, TIME-02, TIME-03)

<domain>
## Phase Boundary

Add a 12th workspace tab "Time" to every customer page. The tab shows a log of time entries for the project (date, hours, description), a total hours summary, and a date range filter. The user can add, edit, and delete entries inline or via a simple form. All entries are exportable as CSV (date, hours, description, project name). No complex billing or invoice features — this is a personal time log.

</domain>

<decisions>
## Implementation Decisions

### Tab & Navigation
- 12th tab label: "Time" — added to the workspace tab nav alongside existing tabs
- Route: `/customer/[id]/time` with a new `time/page.tsx`
- Tab nav file: `bigpanda-app/app/customer/[id]/layout.tsx` (add "Time" entry)

### Database
- New table: `time_entries (id, project_id, date date, hours numeric(5,2), description text, created_at, updated_at)`
- Migration: `0006_time_tracking.sql` — written manually (no drizzle-kit)
- `hours` stored as `numeric(5,2)` — supports decimal (e.g., 1.5 hours)
- Date defaults to today when adding a new entry

### API
- `GET /api/projects/[projectId]/time-entries` — list all entries, ordered by date DESC
- `POST /api/projects/[projectId]/time-entries` — create entry (date, hours, description)
- `PATCH /api/projects/[projectId]/time-entries/[entryId]` — update entry
- `DELETE /api/projects/[projectId]/time-entries/[entryId]` — delete entry
- All use plain `fetch() + useState + useEffect` pattern (no TanStack Query)

### UI
- Table view: columns = Date | Hours | Description | Actions (Edit / Delete)
- Header row: total hours (sum of filtered entries)
- Date range filter: "From" and "To" date inputs — client-side filter, no server round-trip
- Add entry: inline form at top of table OR "+ Log Time" button that reveals a small inline form (no modal — keep it simple)
- Edit: click pencil icon on row → row becomes editable inline
- Delete: click trash icon → immediate delete (no confirmation needed — entries are trivially re-addable)
- Hours input: decimal number input with step=0.25
- CSV export: client-side (generate Blob in browser, no server route) — filename `time-log-{projectName}-{date}.csv`

### What is NOT in scope
- Invoice generation
- Billing rates or cost calculation
- Multi-user time tracking (single user app)
- Integration with external time tracking tools
- Phase 6 MCP integrations

</decisions>

<specifics>
## Specific Ideas

- Total hours display: `X.X hrs total` above the table, updates as date filter changes
- Date filter: two `<input type="date">` fields — filter applied client-side on the existing `entries` state
- "Log Time" form fields: Date (default today), Hours (numeric, step 0.25, min 0.25), Description (text, required)
- CSV columns: Date, Hours, Description, Project (project name from context)
- Export filename: `time-log-AMEX-2026-03-23.csv`
- Empty state: "No time logged yet — click 'Log Time' to add your first entry"

</specifics>

- Invoice PDF export
- External integrations (Harvest, Toggl, Jira)

</deferred>
