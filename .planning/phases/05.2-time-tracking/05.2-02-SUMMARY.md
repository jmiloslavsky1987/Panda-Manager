## Decisions Made

- `hours` stored as TEXT (not NUMERIC) — consistent with project-wide schema convention; JS `parseFloat()` handles arithmetic
- `date` stored as TEXT (not DATE type) — consistent with all other date fields in the schema
- No RLS policy on `time_entries` — single-user app; plain `WHERE project_id = N` is sufficient (established in Phase 5.1)
- Migration written manually — `drizzle-kit generate` not available in this project (decision from Phase 01-02)
