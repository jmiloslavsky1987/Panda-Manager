**bigpanda-app/components/TimeEntryModal.tsx**

Dual-mode modal (create when no `entry` prop; edit when `entry` prop provided). Uses shadcn Dialog.

Props interface:
```typescript
interface TimeEntryModalProps {
  projectId: number
  entry?: TimeEntry          // undefined = create, defined = edit
  trigger: React.ReactNode
  onSuccess: () => void      // parent re-fetches on success
}
```

Form fields:
- Date: `<input type="date">` — defaults to today (ISO: `new Date().toISOString().slice(0,10)`) in create mode, or `entry.date` in edit mode
- Hours: `<input type="number" step="0.25" min="0.25">` — defaults to `''` in create, or `entry.hours` in edit
- Description: `<input type="text">` — defaults to `''` in create, or `entry.description` in edit

On submit:
- Create mode: `POST /api/projects/${projectId}/time-entries` with `{ date, hours: hoursStr, description }`
- Edit mode: `PATCH /api/projects/${projectId}/time-entries/${entry.id}` with `{ date, hours: hoursStr, description }`
- On success: close dialog, call `onSuccess()`
- On error: show error message in form

Hours value: send as string (e.g. `String(hoursValue)`) to match API validation (TEXT column, Zod string check).

Total hours: `entries.reduce((sum, e) => sum + parseFloat(e.hours), 0).toFixed(2)` — use `toFixed(2)` on the sum only.

CSV export function (per RESEARCH.md Pattern 6):
```typescript
function exportCSV(entries: TimeEntry[], projectName: string) {
  const header = 'date,hours,description,project name'
  const rows = entries.map(e =>
    [e.date, e.hours, `"${e.description.replace(/"/g, '""')}"`, `"${projectName}"`].join(',')
  )
  const csv = [header, ...rows].join('\n')
  const blob = new Blob([csv], { type: 'text/csv' })
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = `time-log-${projectName}-${new Date().toISOString().slice(0,10)}.csv`
  a.click()
  URL.revokeObjectURL(url)
}
```
