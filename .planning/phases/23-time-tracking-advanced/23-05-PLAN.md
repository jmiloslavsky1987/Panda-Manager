  <name>Task 1: Bulk operations API route</name>
  <files>bigpanda-app/app/api/projects/[projectId]/time-entries/bulk/route.ts</files>
  <action>
    Create POST /api/projects/[projectId]/time-entries/bulk:

    Request body schema (zod):
    ```typescript
    z.object({
      action: z.enum(['approve', 'reject', 'move', 'delete']),
      entry_ids: z.array(z.number()).min(1).max(100),
      // action-specific fields:
      approved_by: z.string().optional(),      // for approve
      rejected_by: z.string().optional(),      // for reject
      reason: z.string().optional(),           // for reject (required if action=reject)
      target_project_id: z.number().optional(), // for move (required if action=move)
    })
    ```

    Implement each action inside a DB transaction (with RLS SET LOCAL):

    approve:
    - Filter entry_ids to only those with submitted_on set (skip non-submitted — do not error, just skip)
    - Fetch lock_after_approval from time_tracking_config
    - Bulk update: approved_on=NOW(), approved_by, locked=lock_after_approval using .returning()
    - After the transaction, loop over returned updated entries and call buildApprovalNotification(entry, approved_by) for each:
      ```typescript
      import { buildApprovalNotification } from '@/lib/time-tracking-notifications'
      // Outside the transaction (notifications are best-effort — do not roll back approval on notify failure):
      for (const entry of updatedEntries) {
        await buildApprovalNotification(entry, approved_by).catch(err =>
          console.error('Notification failed for entry', entry.id, err)
        );
      }
      ```
    - Return { processed: N, skipped: M }

    reject:
    - Validate reason is present (400 if missing)
    - Filter to submitted entries only
    - Bulk update: rejected_on=NOW(), rejected_by, description appended with "[Rejected: reason]"
    - Return { processed: N, skipped: M }

    move:
    - Validate target_project_id present (400 if missing)
    - Filter to canEdit entries (draft or rejected only — cannot move submitted/approved/locked)
    - Bulk update: project_id=target_project_id
    - Return { processed: N, skipped: M }

    delete:
    - Validate entries are not locked (skip locked — cannot delete without unlock)
    - Bulk delete using inArray(timeEntries.id, validIds)
    - Write audit log: action='bulk_delete', entity_type='time_entry_batch', before_json={ids: validIds}
    - Return { deleted: N, skipped: M }

    Write audit log for all actions with entity_type='time_entry_batch'.
  </action>
