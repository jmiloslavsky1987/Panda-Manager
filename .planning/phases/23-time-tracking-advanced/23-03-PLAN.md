  <action>
    Create submit route (POST /api/projects/[projectId]/time-entries/submit):
    - Accepts body: { week_start: string (YYYY-MM-DD), submitted_by?: string }
    - Fetches all draft entries for projectId in the specified week (date between week_start and week_start+6 days)
    - Validates there are entries to submit (400 if none)
    - Sets submitted_on=NOW(), submitted_by=(body.submitted_by ?? 'user') on all matching draft entries
    - Writes audit log entry: action='submit', entity_type='time_entry_batch', before/after JSON
    - Returns { submitted_count: N, entries: [...] }

    Create approve route (POST /api/projects/[projectId]/time-entries/[entryId]/approve):
    - Accepts body: { approved_by?: string }
    - Validates entry exists and is in 'submitted' status (canSubmit=false, approved_on=null, rejected_on=null)
    - Fetches time_tracking_config (id=1) to check lock_after_approval
    - Updates: approved_on=NOW(), approved_by=(body.approved_by ?? 'approver'), locked=(lock_after_approval)
    - Writes audit log: action='approve'
    - Returns updated entry

    Create reject route (POST /api/projects/[projectId]/time-entries/[entryId]/reject):
    - Accepts body: { rejected_by?: string, reason: string }
    - Validates reason is present (400 if missing — TTADV-08 requires reason)
    - Validates entry is in 'submitted' status
    - Updates: rejected_on=NOW(), rejected_by=(body.rejected_by ?? 'approver'), description=`${entry.description} [Rejected: ${reason}]`
    - Note: do NOT set locked — rejected entries return to 'draft' status and can be edited/resubmitted
    - Writes audit log: action='reject'
    - Returns updated entry
