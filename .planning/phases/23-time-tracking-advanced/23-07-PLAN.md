    ```sql
    CREATE TABLE IF NOT EXISTS "app_notifications" (
      "id"          SERIAL PRIMARY KEY,
      "user_id"     TEXT NOT NULL DEFAULT 'default',   -- single-user for now; extends naturally for multi-user
      "type"        TEXT NOT NULL,                      -- 'timesheet_reminder' | 'timesheet_overdue' | 'timesheet_approved' | 'timesheet_rejected'

      "body"        TEXT NOT NULL,

      "data"        JSONB,                              -- extra context: { project_id, entry_ids, reason }
      "created_at"  TIMESTAMP DEFAULT NOW() NOT NULL
    );
    CREATE INDEX IF NOT EXISTS idx_app_notifications_user ON app_notifications(user_id, read, created_at DESC);
    ```

    Add appNotifications table to bigpanda-app/db/schema.ts after timeTrackingConfig.

    Create bigpanda-app/lib/time-tracking-notifications.ts:

    ```typescript
    import db from '@/db'
    import { appNotifications, timeEntries, timeTrackingConfig } from '@/db/schema'
    import { eq, and, isNull } from 'drizzle-orm'
    import type { TimeEntry } from '@/db/schema'

    // Determine which users have unsubmitted entries for the current week
    // Returns array of user notifications to create (excludes exempt_users)
    export async function computePendingReminders(): Promise&lt;{ title: string; body: string; type: string }[]&gt;

    // Write a single approval notification to DB
    export async function buildApprovalNotification(entry: TimeEntry, approvedBy: string): Promise&lt;void&gt;

    // Write a single rejection notification to DB with reason
    export async function buildRejectionNotification(entry: TimeEntry, rejectedBy: string, reason: string): Promise&lt;void&gt;
    ```

    computePendingReminders logic:
    1. Fetch time_tracking_config (enabled check — skip entirely if enabled=false)
    2. Compute current week bounds (Mon-Sun ISO dates)
    3. Find all projects with at least one time entry this week that has submitted_on=null
    4. For each such project, build a reminder notification: "Timesheet due {due_day} at {due_time} — {N} draft entries for {project}"
    5. Exclude users in exempt_users list (user_id='default' for now — skip exempt check on single user)
    6. Insert notifications into app_notifications (type='timesheet_reminder')
    7. Return count

    buildApprovalNotification:
    - Insert: type='timesheet_approved', title='Time Entry Approved', body=`Your entry "${entry.description}" on ${entry.date} has been approved by ${approvedBy}`

    buildRejectionNotification:
    - Insert: type='timesheet_rejected', title='Time Entry Rejected', body=`Your entry "${entry.description}" on ${entry.date} was rejected by ${rejectedBy}. Reason: ${reason}`

    Apply migration: `psql $DATABASE_URL -f db/migrations/0016_notifications.sql`
