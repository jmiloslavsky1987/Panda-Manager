## Decisions Made
- `prompt: 'consent'` is required in `generateAuthUrl` — without it, Google only returns `refresh_token` on first authorization. Subsequent connects (after revoking or reconnecting) would not return a `refresh_token`, breaking the adapter permanently.
- Email is extracted from `id_token` using `verifyIdToken` — this is a nice-to-have for display in Settings. Non-fatal if absent or if verification fails.
- `user_id = 'default'` keeps the schema forward-compatible with multi-user auth. The UNIQUE(user_id, source) constraint enables single-row upsert per user per source.

## User Setup Required

**External services require manual configuration before Gmail OAuth can be used:**

1. Create a Google Cloud project with OAuth 2.0 credentials
2. Enable Gmail API: Google Cloud Console → APIs & Services → Library → Gmail API → Enable
3. Add Gmail readonly scope: OAuth consent screen → Scopes → `https://www.googleapis.com/auth/gmail.readonly`
4. Create OAuth 2.0 Client ID credentials
5. Add to `.env.local`:
   ```
   GOOGLE_CLIENT_ID=<your-client-id>
   GOOGLE_CLIENT_SECRET=<your-client-secret>
   GOOGLE_REDIRECT_URI=http://localhost:3001/api/oauth/gmail/callback
   ```
6. Add `http://localhost:3001/api/oauth/gmail/callback` to Authorized redirect URIs in the OAuth client

See 19.1-VALIDATION.md SRC-05 for end-to-end verification steps.
