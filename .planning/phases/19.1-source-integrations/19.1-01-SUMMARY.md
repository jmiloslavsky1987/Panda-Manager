## Decisions Made
- `SourceCredentials` is defined in `settings-core.ts` rather than `source-adapters/index.ts` because `index.ts` imports `MCPServerConfig` from `settings-core.ts` — placing credentials in the adapter module would create a circular import
- `resolveAdapter` stub returns `null` for all inputs — this is intentional; the factory logic requires all adapter implementations to exist first (Plan 19.1-05 turns these RED→GREEN)
- `user_source_tokens` uses `UNIQUE(user_id, source)` to enforce one OAuth token per source per user, using `DEFAULT 'default'` for user_id to stay single-user-compatible while being forward-compatible with future auth
