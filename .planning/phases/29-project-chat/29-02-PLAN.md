Note on initialContext delivery: The chat route handler (Plan 01) calls buildChatContext(projectId) internally on the server for the system prompt. The page server component also calls buildChatContext to pass as a prop for any client-side display needs. The API route is the authoritative source — context in the route handler system prompt is what matters for CHAT-02. The ChatPanel's initialContext prop is passed to the route via an extra body field so the route can optionally use it (avoids double DB query on first message — see 29-RESEARCH.md Pitfall 2).

Alternatively (simpler): The page server component pre-fetches context and passes it as initialContext to ChatPanel. ChatPanel adds it to each request body as `{ messages, context: initialContext }`. The route handler uses the passed context instead of re-querying DB. This is the preferred approach per RESEARCH.md (pre-load on mount).
