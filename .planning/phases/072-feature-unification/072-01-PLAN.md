<objective>
Add pgEnum types for Risks and Milestones status columns in the database schema and migrate existing text columns to use them. This closes the highest-priority finding from the Phase 71 audit: invalid status values can reach the database because `risks.status` and `milestones.status` are plain `text` with no DB-level constraint.

Purpose: Enforce data integrity at the database layer — the same protection that `actionStatusEnum` already provides for Actions.
Output: Two new pgEnum definitions in schema.ts, a numbered forward-only SQL migration file, and updated migrate-local.ts validation.
</objective>

<task type="auto">
  <name>Task 2: Generate migration and write 0036 SQL with data normalization</name>
  <files>bigpanda-app/db/migrations/0036_risk_milestone_status_enums.sql</files>
  <action>
Run `cd bigpanda-app && npx drizzle-kit generate` to generate the base SQL migration. The generated file will be in db/migrations/ — rename it to `0036_risk_milestone_status_enums.sql` if drizzle-kit generates a different name.

Then edit the generated SQL to add data normalization UPDATE statements BEFORE the ALTER COLUMN statements. The final migration must follow this order:

1. Create the enum types:
```sql
CREATE TYPE "public"."risk_status" AS ENUM('open', 'mitigated', 'resolved', 'accepted');
CREATE TYPE "public"."milestone_status" AS ENUM('not_started', 'in_progress', 'completed', 'blocked');
```

2. Normalize existing data (BEFORE adding enum constraint):
```sql
-- Normalize risks.status: null stays null, valid values stay, anything else → null
UPDATE "risks" SET "status" = CASE
  WHEN "status" IN ('open', 'mitigated', 'resolved', 'accepted') THEN "status"
  ELSE NULL
END WHERE "status" IS NOT NULL AND "status" NOT IN ('open', 'mitigated', 'resolved', 'accepted');

-- Normalize milestones.status: null stays null, valid values stay, anything else → 'not_started'
UPDATE "milestones" SET "status" = CASE
  WHEN "status" IN ('not_started', 'in_progress', 'completed', 'blocked') THEN "status"
  ELSE 'not_started'
END WHERE "status" IS NOT NULL AND "status" NOT IN ('not_started', 'in_progress', 'completed', 'blocked');
```

3. Alter columns to use the new enum types:
```sql
ALTER TABLE "risks" ALTER COLUMN "status" SET DATA TYPE "public"."risk_status" USING "status"::"public"."risk_status";
ALTER TABLE "milestones" ALTER COLUMN "status" SET DATA TYPE "public"."milestone_status" USING "status"::"public"."milestone_status";
```

This is a forward-only migration (no rollback) — matching the existing project pattern.

IMPORTANT: Do NOT run the migration against the database in this task. The migration file is the deliverable.
