#### 1. OVER-04 test with populated DB

**Test:** Seed at least one onboarding phase + step using `npx tsx bigpanda-app/scripts/import-onboarding.ts`, then run `npx playwright test tests/e2e/phase5.1.spec.ts --grep "OVER-04"`.

**Why human:** The assert-if-present guard (`if (phaseCount > 0)`) skips the entire interaction block on an empty DB. Both fixes (endpoint path + click-before-fill) are only exercised with seeded data.

#### 2. Sticky header project summary with populated DB

**Test:** Load `/customer/1/overview` in a browser with a project row that has `status_summary` and `go_live_target` populated.
**Expected:** Customer name appears in the sticky header (bold, truncated). Executive summary text renders below step count (2-line clamp). Go-live badge renders with the correct date string.
**Why human:** All three fields are conditionally rendered — they require non-null DB values. The OVER-01 E2E test only checks structural testids and will pass even with null fields.

#### 3. Animated progress ring

**Test:** Load `/customer/1/overview` in a browser with steps seeded in the DB (some marked complete).
**Expected:** The SVG ring animates from 0% to the actual completion percentage over 1.5s.
**Why human:** CSS animation (`transition: stroke-dashoffset 1.5s ease`) cannot be verified in headless Playwright.

#### 4. Collapsible phase cards

**Test:** Load the overview page with at least one fully-complete phase and one in-progress phase.
**Expected:** Fully-complete phase is collapsed by default; in-progress phase is open. Clicking the header toggles the state.
**Why human:** Collapse state and animation smoothness require interactive browser inspection.
