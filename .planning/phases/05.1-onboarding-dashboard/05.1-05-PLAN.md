  <action>
Create `bigpanda-app/scripts/import-onboarding.ts`. The script accepts a `--project-id` arg and a `--file` arg (path to the YAML context doc). It reads the `onboarding_phases` and `integrations` keys from the YAML frontmatter and inserts them into the DB using `onConflictDoNothing()`.

**Script structure:**

```typescript
#!/usr/bin/env tsx
// Usage: npx tsx scripts/import-onboarding.ts --project-id 1 --file ~/Documents/PM Application/Kaiser_context.md

import { db } from '../db/index'
import { onboardingPhases, onboardingSteps, integrations } from '../db/schema'
import yaml from 'js-yaml'
import fs from 'fs'
import path from 'path'

async function main() {
  const args = process.argv.slice(2)
  const projectIdArg = args[args.indexOf('--project-id') + 1]
  const fileArg = args[args.indexOf('--file') + 1]
  if (!projectIdArg || !fileArg) {
    console.error('Usage: npx tsx scripts/import-onboarding.ts --project-id N --file /path/to/context.md')
    process.exit(1)
  }
  const projectId = parseInt(projectIdArg, 10)
  const filePath = fileArg.replace('~', process.env.HOME ?? '')

  const raw = fs.readFileSync(filePath, 'utf-8')
  const fmMatch = raw.match(/^---\n([\s\S]*?)\n---/)
  if (!fmMatch) { console.error('No YAML frontmatter found'); process.exit(1) }
  const doc = yaml.load(fmMatch[1], { schema: yaml.JSON_SCHEMA }) as Record<string, unknown>

  const obPhases = (doc.onboarding_phases as any[]) ?? []
  const integsList = (doc.integrations as any[]) ?? []

  // Insert phases and steps
  for (const phase of obPhases) {
    const [inserted] = await db.insert(onboardingPhases)
      .values({ project_id: projectId, name: phase.name, display_order: phase.display_order ?? 0 })
      .onConflictDoNothing()
      .returning({ id: onboardingPhases.id })
    if (!inserted) { console.log(`Skipping existing phase: ${phase.name}`); continue }
    const phaseId = inserted.id
    for (const step of phase.steps ?? []) {
      await db.insert(onboardingSteps).values({
        phase_id: phaseId,
        project_id: projectId,
        name: step.name,
        description: step.description ?? null,
        status: step.status ?? 'not-started',
        owner: step.owner ?? null,
        dependencies: step.dependencies ?? [],
        updates: [],
        display_order: step.display_order ?? 0,
      }).onConflictDoNothing()
    }
  }

  // Insert integrations
  for (const integ of integsList) {
    await db.insert(integrations).values({
      project_id: projectId,
      tool: integ.tool,
      category: integ.category ?? null,
      status: integ.status ?? 'not-connected',
      color: integ.color ?? null,
      notes: integ.notes ?? null,
      display_order: integ.display_order ?? 0,
    }).onConflictDoNothing()
  }

  console.log(`Imported ${obPhases.length} phases, ${integsList.length} integrations for project ${projectId}`)
  process.exit(0)
}

main().catch(e => { console.error(e); process.exit(1) })
```

Note: `onConflictDoNothing()` on `onboardingPhases` will conflict on the primary key (serial id). Since serial IDs auto-increment, the conflict target is effectively a no-op that won't actually prevent duplicates unless a UNIQUE constraint exists. For idempotency without a UNIQUE constraint, check if the phase already exists before inserting:

```typescript
// Before insert, check existence:
const existing = await db.select().from(onboardingPhases)
  .where(and(eq(onboardingPhases.project_id, projectId), eq(onboardingPhases.name, phase.name)))
if (existing.length > 0) { console.log(`Skipping: ${phase.name}`); continue }
// Then insert normally (no onConflictDoNothing needed)
```

Use the existence-check approach for robust idempotency (RESEARCH.md notes onConflictDoNothing works on explicit UNIQUE constraints — check if migration adds any; if not, use the existence-check pattern).

**Route logic:**

1. Parse `projectId` from params (isNaN guard → 400)
2. Load project from DB to get `source_file` field
3. Load `settings.workspace_path` from `readSettings()` (import from `@/lib/settings`)
4. Construct full file path: `path.join(settings.workspace_path, project.source_file)`
5. Check file exists (if not → 404 with `{ error: 'Context file not found' }`)
6. Read file, extract frontmatter with regex `/^---\n([\s\S]*?)\n---/`
7. Parse frontmatter with `parseYaml()` (from `lib/yaml-export.ts` at project root — import path is `../../../../lib/yaml-export` from bigpanda-app/app/api/)
8. Query current DB state in RLS transaction:
   - phases with steps (same query as onboarding GET route)
   - integrations (same query as integrations GET route)
9. Serialize DB state into YAML-compatible objects:
   ```typescript
   const yamlPhases = phases.map(p => ({
     name: p.name,
     display_order: p.display_order,
     steps: p.steps.map(s => ({
       name: s.name, description: s.description, status: s.status,
       owner: s.owner, dependencies: s.dependencies, display_order: s.display_order,
     }))
   }))
   const yamlIntegrations = integrations.map(i => ({
     tool: i.tool, category: i.category, status: i.status,
     color: i.color, notes: i.notes, display_order: i.display_order,
   }))
   ```
10. Mutate the parsed YAML doc object (in-place, NOT rebuilding the whole doc):
    ```typescript
    doc.onboarding_phases = yamlPhases
    doc.integrations = yamlIntegrations
    ```
11. Serialize with `serializeProjectToYaml(doc)` — uses non-negotiable settings
12. Write back: `---\n${newYaml}---\n\n${bodyAfterFrontmatter.trimStart()}`
13. Return `{ ok: true }`
