## Decisions Made
- JSONB append uses `sql\`${onboardingSteps.updates} || ${JSON.stringify([newEntry])}::jsonb\`` — plain `.set({ updates: value })` would overwrite the existing array, losing all prior update history
- All routes use `@/db` alias imports as required by must_haves constraint — avoids fragile relative paths at deep nesting levels
- RLS `SET LOCAL` used (not `SET`) — scopes to transaction preventing context bleed across connection pool
