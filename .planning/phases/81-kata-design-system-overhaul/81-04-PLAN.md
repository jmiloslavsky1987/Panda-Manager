    Open lib/queries.ts and ADD two new exported query functions. Do NOT modify existing functions.

    **getPortfolioWeekMetrics({ userId, isGlobalAdmin }):**
    - Returns { tasksClosedThisWeek: number, milestonesHitThisWeek: number, overdueTasks: number, updatesLogged: number }
    - tasksClosedThisWeek: tasks with status='completed' and updated_at in last 7 days, filtered by user's projects
    - milestonesHitThisWeek: milestones with status='Complete' and updated_at in last 7 days
    - overdueTasks: tasks with due_date < today and status != 'completed', across user's projects
    - updatesLogged: count of engagement_history entries in last 7 days across user's projects
    - Wrap in try/catch — return zeros on error (graceful degradation)

    **getPortfolioBriefingData({ userId, isGlobalAdmin }):**
    - Returns { upcomingGoLives: ProjectGoLive[], highSeverityRisks: RiskEntry[], needsAttention: AttentionProject[] }

    upcomingGoLives: Projects from user's accessible projects where go_live_target is between today and +28 days.
    Return: { id: number, customer: string, go_live_target: string }[]

    highSeverityRisks: Risks with status='open' (or no closed status) AND likelihood='High' OR impact='High'
    across user's accessible projects. Return: { id: number, title: string, projectCustomer: string }[]
    Note: Check existing risks table schema for column names (likelihood, impact fields added in Phase 76).

    needsAttention: Projects with health='red' OR no engagement_history entry with created_at > 7 days ago.
    Return: { id: number, customer: string, reason: 'red-health' | 'stale' }[]
    De-duplicate: if a project is both red AND stale, return it once with reason='red-health'.

    All three queries must respect userId/isGlobalAdmin scoping (same pattern as getPortfolioData).
    Wrap each in try/catch returning empty arrays on error.
