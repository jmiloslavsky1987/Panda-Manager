## Problem

Phase 30 verification testing is a high-level end-to-end smoke test. A thorough UAT pass across all workspace tabs has not been done — there are likely many small bugs, UX issues, edge cases, and missing polish items that have accumulated across v1.0–v3.0 phases.

## Solution

User to go through each workspace tab systematically and document all changes/fixes wanted. Expected to produce a significant backlog of items. Process:

1. Open each tab in the workspace for a real project
2. Document all issues, inconsistencies, missing behavior, UX friction
3. Collect into a fixes backlog (likely a new phase or set of phases)
4. Prioritize and plan as a dedicated polish/bug-fix milestone

Tabs to cover: Overview, Plan (Board/Gantt/Swimlane/Tasks), Actions, Risks, Decisions, Milestones, History, Stakeholders, Teams, Architecture, Skills, Time, Queue, Context, Chat, Scheduler, Settings.

Also covers Reports and any other output surfaces.
