## Integration Findings

### Critical Gaps

**[CRITICAL] Scheduler gap — morning-briefing and weekly-customer-status missing from JOB_SCHEDULE_MAP**

`bigpanda-app/worker/scheduler.ts` defines `JOB_SCHEDULE_MAP` with 6 entries. `morning-briefing` and `weekly-customer-status` are absent — these jobs will never fire on a schedule. Both handlers are fully implemented and registered in `JOB_HANDLERS` but require manual `/api/jobs/trigger` calls.

- Breaks: SCHED-01 (morning briefing 8am), SCHED-03 (weekly status Thursday 4pm), SKILL-11 (morning briefing auto-run), SKILL-03 (weekly customer status scheduled), DASH-01 (briefing panel auto-populated), DASH-09 (Drafts Inbox auto-populated by weekly status)
- File: `bigpanda-app/worker/scheduler.ts` lines 9–16
- Side note: `action-sync` key maps to `morning_briefing` schedule slot — likely a naming mismatch; `action-sync` runs xlsx sync, not the morning briefing skill

### Moderate Gaps

**[MODERATE] YAML export route unreachable from UI (DATA-05, OVER-04)**

`/api/projects/[projectId]/yaml-export` is fully implemented and wired to `serializeProjectToYaml()`. Zero callers found in the UI layer. No Export YAML button exists anywhere in the workspace. Users cannot trigger YAML export without direct API calls.

- File: `bigpanda-app/app/api/projects/[projectId]/yaml-export/route.ts`
- Needed: Add "Export YAML" button to `ProjectHeader.tsx` or workspace layout

**[MODERATE] SET-02 skill path not respected by 3 scheduled job handlers (SET-02, SKILL-14)**

`morning-briefing.ts`, `weekly-customer-status.ts`, and `context-updater.ts` hardcode `path.join(__dirname, '../../skills')`. Only `skill-run.ts` uses `resolveSkillsDir(settings.skill_path)` introduced in Phase 10. Custom `skill_path` in Settings is ignored for scheduled/named handlers.

- Files: `bigpanda-app/worker/jobs/morning-briefing.ts:14`, `weekly-customer-status.ts:14`, `context-updater.ts:15`

### Low Priority

**[LOW] Search filter TYPE_OPTIONS missing 4 FTS tables (SRCH-02, SRCH-03)**

`app/search/page.tsx` TYPE_OPTIONS only includes 8 of 12 searchable tables. Phase 10 expanded FTS to `onboarding_steps`, `onboarding_phases`, `integrations`, `time_entries` in the backend query but the filter dropdown was not updated. All-types search works; per-type filtering cannot target the 4 new tables.

- File: `bigpanda-app/app/search/page.tsx` lines 18–28
